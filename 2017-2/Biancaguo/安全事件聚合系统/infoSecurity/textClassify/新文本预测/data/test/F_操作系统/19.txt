

OS X 的「定位」是 laptop - workstation 的操作系统。所谓「定位」，我理解为「设计者主力想做，而且做得很好的目标」。OS X 也能装在高端 Mac Pro 上当 server，各种 Linux distro （注意，Android, Chromium OS 都不是一般意义的 Linux distro） 也能做 laptop 操作系统，但是那叫「兼顾」（或者叫硬弩劲），不叫「定位」。Laptop - workstation 的特点是同一时间只主要服务一个用户（但是今天的操作系统都是多用户系统）。OS X 的优势就是它的定位。因为在这个定位领域中，没有其它的 Unix vendor。现在我们来看 Unix 的定位。广义的 Unix 是一种操作系统流派，主要由 POSIX 标准来制定（system call，主要命令行工具，进程/线程行为，文件系统，等等）。Unix 流派是 Unix vendor 来设计完整的操作系统的基石。所以 Unix 的「定位」是「通用的（从嵌入式系统到分布式集群和大型机）操作系统基础」。从历史来看，Unix 的定位是成功的，也就是说，我们的世界里就不应该有非 Unix 的系统存在（除非严格的实时操作系统，比如火星车降落控制级别，但是经过调整的 Unix 可以满足绝大多数比火星车稍宽松的实时需求）！在 Windows 里除了 GUI 系统之外最成功的部分也是借鉴 Unix 最彻底的部分。当然，窗口系统并不是 Unix 流派强行规定的部分。所以，很可惜，在 OS X 之前，Unix 这么一个从理论上到实践上都被不断的独立证明适合一切领域的系统，偏偏在 laptop - workstation 中没有一个真正完整的系统（当然，在 workstation 还是大实验室设备的时候，各大 Unix vendor 还是短暂的竞争过一段时间）。
